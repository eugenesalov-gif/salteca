# Salteca

**Salteca** — утилита для macOS, которая на лету исправляет текст, набранный
в неправильной раскладке (`ghbdtn` → `привет`, `руддщ` → `hello`), и переключает
системную раскладку под исправленный результат, чтобы вы продолжили печатать
уже правильно.

Живёт в строке меню (menu bar), без иконки в Dock.

## О проекте

Проект вырос из личной боли: постоянно ловишь себя на том, что набрал целую
фразу, не переключив раскладку, — и приходится стирать, переключаться, набирать
заново. Существующие решения либо платные, либо тяжёлые, либо не переключают
саму системную раскладку следом за исправлением. Хотелось лёгкий menu-bar-инструмент,
который делает ровно одно и делает это незаметно.

Это порт и развитие моего же Python-прототипа на нативный Swift — ради скорости,
надёжной работы с системными API и распространения одним `.app`.

## Функции

- **Авто-режим** — исправляет слова прямо по мере набора, в фоне, без хоткея.
  Слово завершается пробелом/Enter/пунктуацией → раскладка проверяется и, если
  надо, текст переписывается на месте.
- **Хоткей-режим** — исправляет выделенное или последнее слово по глобальному
  хоткею (по умолчанию ⇧⌘X). Повторный хоткей на уже исправленном слове
  откатывает правку обратно.
- **Настраиваемый хоткей** — комбинацию можно переназначить в настройках.
- **Звук переключения** — короткий сигнал ровно в момент *подтверждённой* смены
  раскладки (можно выбрать звук или отключить).
- **Автозапуск при входе** — регистрируется через `SMAppService`; источник
  правды — сама система, чекбокс всегда показывает реальное состояние.

## Технические детали / архитектура

Стек: **Swift / SwiftUI**, **Carbon Text Input Sources (TIS) API** для раскладок,
**Accessibility (AX) API** и **CGEvent** для чтения/замены текста, `SMAppService`
для автозапуска. Без сторонних зависимостей.

Несколько решений, которыми проект интересен как инженерная задача:

- **Carbon TIS только с главного потока.** `TISCreateInputSourceList` /
  `TISSelectInputSource` обязаны вызываться с main thread — иначе
  `dispatch_assert_queue_fail` и краш. Захват текста при этом идёт на фоновой
  serial-очереди, поэтому весь поиск и переключение раскладки выполняются одним
  хопом на main (`DispatchQueue.main.sync`, безопасным здесь, т.к. TIS-вызовы
  быстрые и не прокручивают run loop).

- **Переключение подтверждается уведомлением, а не таймером.** Раньше после
  смены раскладки ждали фиксированную паузу (~0.5 с) — это и медленно, и
  ненадёжно: пользовательские нажатия попадали в момент незавершённого
  переключения, и tap мисчитывал символы. Теперь код блокируется на семафоре до
  распределённого уведомления `kTISNotifySelectedKeyboardInputSourceChanged` (с
  коротким failsafe-таймаутом). Звук играет ровно по подтверждению. Чужие
  переключения (например, Cmd+Space вручную) игнорируются — сигналим только на
  смену к нашей цели.

- **«Документ заморожен на время правки».** В авто-режиме backspace+paste
  физически меняют документ ~0.3 с. Всё это время реальные нажатия пользователя
  подавляются на event tap и складываются в очередь, а после завершения правки
  «дотипливаются» в правильную позицию и снова прогоняются через движок (каскадные
  правки). Так модель движка и реальный документ никогда не расходятся, а
  сегментация слов не зависит от таймингов.

- **Race conditions и settle-паузы.** Синтетические стрелки/копирование требуют
  микропауз, чтобы целевое приложение успело применить выделение до чтения. Хоткей
  ещё физически зажат в момент запуска — ждём отпускания модификаторов, иначе
  Cmd+C превращается в Cmd+Shift+C. Настоящий буфер обмена пользователя
  сохраняется и гарантированно восстанавливается через `defer`, а системный «бип»
  глушится на время операции.

- **Чистое ядро под тестами.** Логика определения раскладки, маппинг символов,
  сопоставление источников ввода и конфиг хоткея вынесены в чистые функции и
  покрыты юнит-тестами (`SaltecaTests/`) — включая ловушку, где подстрока `"us"`
  внутри `"russian"` ломала наивное определение английской раскладки.

Общая структура: `AppController` — единый владелец состояния и сервисов (двусторонне
связан с меню/настройками SwiftUI); сервисы (`TextCaptureService`, `AutoModeService`,
`LayoutSwitcher`, `HotKeyManager`) — `Sendable` и работают на своих фоновых потоках.

## Требования

- macOS 26.5 или новее (Apple Silicon).
- Разрешение **Универсальный доступ / Accessibility** (см. ниже) — без него захват
  и замена текста работать не будут.

## Установка

1. Открыть `Salteca-1.0.0.dmg` и перетащить **Salteca.app** в **Applications**.
2. При первом запуске обойти предупреждение Gatekeeper (см. ниже).
3. Выдать доступ в Accessibility (см. ниже).

## Запуск несмотря на предупреждение «неизвестный разработчик»

> **Важно.** У проекта нет платного Apple Developer аккаунта, поэтому приложение
> **не нотаризовано** (notarization требует аккаунта $99/год и сертификата
> Developer ID). Приложение подписано **ad-hoc** — подпись валидна и приложение
> не считается «повреждённым», но Gatekeeper всё равно предупредит, что
> разработчик не подтверждён. Это ожидаемо для распространения вне App Store.

Скачанный `.dmg` получает атрибут карантина, из-за чего срабатывает Gatekeeper.
Обойти — одним из способов:

**Способ 1 — через настройки (рекомендуется, работает на всех версиях):**
1. Попробовать открыть `Salteca.app` (появится предупреждение — закрыть его).
2. **Системные настройки → Конфиденциальность и безопасность**.
3. Внизу найти «Программа "Salteca" была заблокирована…» → **«Открыть всё равно»**,
   подтвердить.

**Способ 2 — правый клик (macOS 14 и старше):**
- Control-клик по `Salteca.app` → **Открыть** → в диалоге снова **Открыть**.

**Способ 3 — через терминал (снять карантин вручную):**
```sh
xattr -dr com.apple.quarantine /Applications/Salteca.app
```

## Требуемые разрешения (Accessibility)

Salteca читает и заменяет текст через системные события клавиатуры, поэтому ей
нужен доступ в Universal Access:

1. **Системные настройки → Конфиденциальность и безопасность → Универсальный
   доступ (Accessibility)**.
2. Включить переключатель напротив **Salteca** (при необходимости добавить через «+»).

> **Нюанс ad-hoc-подписи.** Каждая новая сборка имеет иную подпись. macOS
> привязывает разрешение Accessibility к подписи, поэтому после пересборки может
> потребоваться убрать старую запись Salteca из списка и включить заново.

## Сборка из исходников

Нужен Xcode (Command Line Tools) и [`create-dmg`](https://github.com/create-dmg/create-dmg):

```sh
brew install create-dmg
./scripts/build-dmg.sh
```

Скрипт делает чистый Release-билд, подписывает ad-hoc и упаковывает в
`dist/Salteca-<версия>.dmg`. Версия берётся из `MARKETING_VERSION` проекта.

Запуск тестов:
```sh
xcodebuild test -scheme Salteca -destination 'platform=macOS'
```
